# False Nine

False Nine is a curated archive of modern football: season chapters, core players, club timelines, competitions, transfers, and storylines from 2005 onward.

## How to run locally

Because the data is split into separate JSON files, use a local server.

### VS Code Live Server

1. Open this folder in VS Code.
2. Install the **Live Server** extension.
3. Right-click `index.html`.
4. Click **Open with Live Server**.

### Python local server

```bash
python3 -m http.server 5500
```

Then open:

```txt
http://localhost:5500
```

## URL state

False Nine uses static-friendly hash routes:

```txt
/#season/2008-09
/#player/Lionel%20Messi
/#club/Barcelona
/#about
/#sources
```

This works on GitHub Pages, Netlify, Vercel, Cloudflare Pages, and most static hosts without configuring server rewrites.

## Domain setup

You can point `falsenine.org` at the deployed static site.

Recommended hosts:

- Cloudflare Pages
- Vercel
- Netlify
- GitHub Pages

For Cloudflare Pages / Vercel / Netlify:

1. Put this project in a GitHub repo.
2. Import the repo into the host.
3. Set the build command to blank/none.
4. Set the output/public directory to the project root.
5. Add `falsenine.org` as a custom domain.
6. Follow the host's DNS instructions.

## Data philosophy

This is not trying to be FBref or Transfermarkt.

- No fake detailed stats.
- No autogenerated player bios.
- No engagement bait.
- S Tier means the player was central to that team/season identity.
- A Tier means the player was a major core contributor.
- Core-player tiers are curated context, not exact rankings.

## Adding a season

1. Add a new file to `data/seasons/`, for example `2026-27.json`.
2. Add the matching summary to `data/index.json`.
3. If a new club/nation appears, add a team color mapping in `app.js` and a CSS variable in `style.css` if needed.
4. Test player pages and club pages through search.

The app will automatically pick up the season for browsing, search, player timelines, and club timelines.
